Claude Code

Two ways to connect: MCP server (recommended) or skill file.

Option 1 — Recommended

MCP Server

Gives Claude native tool access to Agent Gateway. No HTTP knowledge needed.

1Get your token

Generate a token right here or on the dashboard.

2Install the MCP server 
npm install -g @tongateway/mcp
3Add to Claude Code settings

Add to .claude/settings.json in your project:

{
  "mcpServers": {
    "agent-gateway": {
      "command": "agent-gateway-mcp",
      "env": {
        "AGENT_GATEWAY_TOKEN": "YOUR_TOKEN_HERE",
        "AGENT_GATEWAY_API_URL": "https://api.tongateway.ai"
      }
    }
  }
}

Paste your token from step 1.

4Use it

Just say "send 1 TON to EQD...abc" and Claude will use the MCP tools automatically.

Option 2

Skill File

A markdown file that teaches Claude the full Agent Gateway API. Drop it into your project.

1Create the skills directory 
mkdir -p .claude/skills
2Download the skill 
curl -o .claude/skills/agent-gateway.md \
  https://tongateway.ai/agent-gateway.md
3Use it

Claude Code will automatically detect the skill. Ask it to /agent-gateway or just say "send TON" and it will know what to do.

Available Tools

ToolParamsDescription
request_transfer to, amountNano, payloadBoc? Queue a TON transfer for owner approval
get_request_status id Check if a request was approved, rejected, or pending
list_pending_requests — List all pending transfer requests

Example

You: Send 0.5 TON to EQD...abc

Agent: I'll request that transfer for you.
       [calls request_transfer with to="EQD...abc", amountNano="500000000"]

       Transfer request created (ID: abc-123).
       Please approve it in your Agent Gateway dashboard.

You: Is it confirmed?

Agent: [calls get_request_status with id="abc-123"]
       Yes, the transfer was confirmed and signed by the wallet.
